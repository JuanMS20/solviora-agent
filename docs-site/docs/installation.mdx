# Installation

## Quick Install (Linux, macOS, WSL2)

Get Solviora Agent up and running with the one-line installer. The
only prerequisite is **Git**. Everything else is handled
automatically — Python, Node.js, ripgrep, ffmpeg, and all dependencies.

```bash
curl -fsSL https://raw.githubusercontent.com/JuanMS20/solviora-agent/main/scripts/install.sh | bash
```

After it finishes, reload your shell and start chatting:

```bash
source ~/.bashrc
solviora setup
solviora
```

:::tip Android / Termux
The installer also works on Android via Termux — it detects the platform
automatically and switches to a tested Android flow with `pkg` for system
dependencies. See the dedicated guide for details.
:::

:::info Windows
Native Windows is not supported. Install [WSL2](https://learn.microsoft.com/en-us/windows/wsl/install)
and run the install command above from your WSL2 terminal.
:::

### What the installer does

The installer handles everything:

- Installs **uv** (fast Python package manager)
- Provisions **Python 3.11** via uv (no sudo needed)
- Clones the Solviora repo to `~/.solviora/solviora-agent/`
- Creates a Python virtual environment and installs all dependencies
- Symlinks the `solviora` command into `~/.local/bin/`
- Adds `~/.local/bin` to your shell's `PATH`
- Syncs bundled skills
- (Optional) Installs Node.js, ripgrep, ffmpeg if missing

### Install layout

| Mode | Code | Command | Data/Config |
|------|------|---------|-------------|
| Per-user | `~/.solviora/solviora-agent/` | `~/.local/bin/solviora` | `~/.solviora/` |
| Root (Linux) | `/usr/local/lib/solviora-agent/` | `/usr/local/bin/solviora` | `/root/.solviora/` |
| Termux | `~/.solviora/solviora-agent/` | `$PREFIX/bin/solviora` | `~/.solviora/` |

## Install with pip (alternative)

```bash
pip install solviora-agent
```

For development, clone the repo and install from source:

```bash
git clone https://github.com/JuanMS20/solviora-agent.git
cd solviora-agent
pip install -e ".[dev]"
```

## Optional dependencies

Solviora ships several optional dependency groups for specific features:

| Group | Install command | What it enables |
|-------|----------------|-----------------|
| `messaging` | `pip install "solviora-agent[messaging]"` | Telegram, Discord, Slack, WhatsApp bots |
| `matrix` | `pip install "solviora-agent[matrix]"` | Matrix/Element bridge |
| `slack` | `pip install "solviora-agent[slack]"` | Slack app |
| `modal` | `pip install "solviora-agent[modal]"` | Modal cloud execution |
| `dev` | `pip install "solviora-agent[dev]"` | Debugging, testing, linting |

You can combine multiple groups:

```bash
pip install "solviora-agent[messaging,dev]"
```

## Shell completion (optional)

To enable tab completion for the `solviora` command:

```bash
# Bash
eval "$(_SOLVIORA_COMPLETE=bash_source solviora)" >> ~/.bashrc

# Zsh
eval "$(_SOLVIORA_COMPLETE=zsh_source solviora)" >> ~/.zshrc

# Fish
_SOLVIORA_COMPLETE=fish_source solviora > ~/.config/fish/completions/solviora.fish
```

## Troubleshooting

### `command not found: solviora`

Make sure your Python `bin` directory is on your `PATH`. With `pip install --user`, that's usually `~/.local/bin` on Linux or `~/Library/Python/3.XX/bin` on macOS.

### Python version too old

Solviora requires Python 3.11 or newer. Check with:

```bash
python --version
```

If your system Python is older, use [pyenv](https://github.com/pyenv/pyenv) or [conda](https://docs.conda.io/) to install a newer version.

:::tip Windows / WSL users
If you're installing on Windows via WSL2, see the [WSL Smoke Test](./wsl-smoke-test) for step-by-step verification.
:::
