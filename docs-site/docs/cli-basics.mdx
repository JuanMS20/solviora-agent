# CLI Basics

The Solviora CLI is an interactive terminal interface for chatting with AI models, running tools, and managing your agent session.

## Starting the CLI

```bash
solviora
```

This opens the interactive prompt with your default model and provider.

## Passing a one-shot prompt

```bash
solviora "Explain async/await in Python"
```

Solviora processes the prompt and exits. Useful for scripting and piping.

## Specifying the model

```bash
solviora --model gpt-4o
solviora --model claude-sonnet-4-20250514
solviora --model gemini-2.5-pro
```

## Interactive commands

Inside the interactive prompt, use slash commands:

| Command | Description |
|---------|-------------|
| `/help` | Show all available commands |
| `/model <name>` | Switch model mid-session |
| `/tools` | List active tools |
| `/skills` | List loaded skills |
| `/history` | Show conversation history |
| `/save` | Save current session |
| `/new` | Start a new session |
| `/compact` | Compress context to stay within limits |
| `/exit` | Exit Solviora |

### Multi-line input

Press **Alt+Enter** or **Ctrl+J** to insert a new line. Essential for pasting code blocks or writing detailed prompts.

### Interrupt the agent

If Solviora is taking too long on a task, type a new message and press Enter. The agent interrupts its current work and switches to your new instructions. **Ctrl+C** also works.

### Personalities

Try a different personality mid-session:

```
/personality pirate
/personality expert
/personality default
```

Personalities are prompt overlays that change the agent's tone. Custom personas can be defined via SOUL.md.

## TUI (terminal UI)

Solviora ships two terminal interfaces:

```bash
solviora              # Classic CLI (prompt_toolkit)
solviora --tui        # Modern TUI with mouse support
```

The TUI (Ink-based) offers modal overlays, mouse selection, and non-blocking input. Both share the same sessions, slash commands, and configuration.

## Sessions

Solviora saves your conversation history automatically. Resume a previous session:

```bash
solviora --resume
```

Or list and pick a session interactively:

```bash
solviora sessions list
```

## Profiles

Run multiple isolated instances with different configurations:

```bash
solviora -p work     # Uses ~/.solviora/profiles/work/
solviora -p personal # Uses ~/.solviora/profiles/personal/
```

Each profile has its own config, API keys, sessions, and skills. Data never crosses profiles.

## Environment variables

| Variable | Purpose |
|----------|---------|
| `SOLVIORA_HOME` | Config directory (default: `~/.solviora`) |
| `OPENAI_API_KEY` | OpenAI API key |
| `ANTHROPIC_API_KEY` | Anthropic API key |
| `GOOGLE_API_KEY` | Google Gemini API key |

:::note
Solviora also reads `HERMES_HOME` for backward compatibility with migrated installations.
:::
