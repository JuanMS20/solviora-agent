# FAQ

## General

### What is Solviora Agent?

Solviora Agent is a terminal-first AI agent that connects to multiple LLM providers (OpenAI, Anthropic, Google, etc.) and uses built-in tools to execute tasks autonomously. It runs in your terminal and can also connect to messaging platforms like Telegram and Discord.

### Is it free?

The agent itself is open source (MIT license). You pay for the LLM API calls you make. You can also use free local providers like Ollama or LM Studio.

### How is it different from ChatGPT / Claude?

Solviora runs locally in your terminal and has direct access to your filesystem, shell, and tools. It can read files, execute code, search the web, and manage scheduled tasks — all from a single conversation.

### Is it related to Hermes Agent?

Yes. Solviora Agent is a fork of Hermes Agent with its own identity, branding, and configuration paths (`~/.solviora/` instead of `~/.hermes/`). It maintains backward compatibility with `HERMES_HOME` for migrated installations.

## Setup

### Which provider should I use?

Start with whichever provider you already have an API key for. OpenAI and Anthropic are the most tested. For free usage, try Ollama with local models.

### Can I use multiple providers at once?

Yes. Use profiles to set up different providers:

```bash
solviora -p openai    # Uses OpenAI config
solviora -p claude    # Uses Anthropic config
```

### Does it work on Windows?

Yes, though WSL (Windows Subsystem for Linux) is recommended for the best experience. Some terminal features work better in a POSIX environment.

## Privacy & Security

### Where is my data stored?

All data stays on your machine in `~/.solviora/`:
- Conversations: `~/.solviora/sessions/`
- Config: `~/.solviora/config.yaml`
- API keys: `~/.solviora/.env`
- Logs: `~/.solviora/logs/`

### Are my API keys sent anywhere?

API keys are only sent to the respective provider's API endpoint. They are never logged, transmitted to third parties, or stored outside `~/.solviora/.env`.

### Can the agent access any file?

The agent operates within your configured `terminal.cwd` directory by default. You can restrict this further in config.

## Features

### Can I use it without tools?

Yes. Run with `--no-tools` for a plain chat mode:

```bash
solviora --no-tools
```

### Can I schedule recurring tasks?

Yes, with the cron system:

```bash
solviora cron add --schedule "0 9 * * *" --prompt "Daily standup summary"
```

### Does it support voice?

Yes, via Edge TTS (free, built-in) or ElevenLabs (premium, requires API key).

### Can I use it from Telegram/Discord?

Yes, via the messaging gateway. See the messaging platform guides in the docs for setup instructions.

```bash
solviora gateway --platform telegram
```

### What's the difference between CLI and TUI?

Solviora ships two terminal interfaces. The classic CLI uses prompt_toolkit for a familiar readline experience. The TUI (Ink-based) adds mouse support, modal overlays, and non-blocking input. Both share the same sessions, slash commands, and config — try each with `solviora` vs `solviora --tui`.

### How do sessions work?

Solviora saves every conversation to `~/.solviora/sessions/` using a SQLite database with FTS5 full-text search. Resume any past session with `solviora --resume` or list them with `solviora sessions list`. Each session maintains lineage tracking across compressions.

### Can I run multiple instances?

Yes, with profiles:

```bash
solviora -p work
solviora -p personal
```

Each profile has its own `~/.solviora/profiles/<name>/` directory with independent config, API keys, sessions, and skills.

### What are skills, and how are they different from plugins?

Skills are prompt-based knowledge documents that teach the agent new workflows without writing code. Plugins are Python packages that add new capabilities (tools, hooks, memory backends) to the agent itself. See the [Skills page](./skills) for details.

### Can I create my own skills?

Yes. Skills follow the SKILL.md format (frontmatter + markdown). You can create them manually or let the agent create them automatically from its experience. Community skills can be installed from the skills hub.

### Does Solviora support MCP?

Yes, Solviora can connect to MCP (Model Context Protocol) servers for extended tool capabilities. MCP server configuration goes in `config.yaml` under `mcp_servers`. *(Advanced feature — see the integration docs for setup details.)*

### How can I customize the agent's personality?

Create a `~/.solviora/SOUL.md` file with your desired voice and behavior. This is injected at the top of the system prompt in every conversation. You can also use the `/personality` slash command to switch built-in personalities mid-session.

### What models does Solviora work with?

Any model with at least 64K tokens of context. Most hosted models (Claude, GPT, Gemini, Qwen, DeepSeek) meet this easily. For local models, set the context size to at least 64K (e.g., `--ctx-size 65536` for llama.cpp).
