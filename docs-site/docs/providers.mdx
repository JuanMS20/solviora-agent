# Providers

Solviora Agent is provider-agnostic. Switch between LLM providers by changing a single config value or command-line flag.

## Supported providers

:::info
Providers below are confirmed in the Solviora codebase. If a provider you need is not listed, try OpenRouter (supports 100+ models) or a custom endpoint.
:::

### Local providers (no API key)

| Provider | Config name | Notes |
|----------|-------------|-------|
| Ollama (local) | `ollama` | Pull a model first: `ollama pull llama3` |
| LM Studio | `lmstudio` | Load a model, server runs at `localhost:1234` |

### OAuth providers

Some providers authenticate via browser OAuth flow. Run `solviora model` and select the provider:

- **Nous Portal** — subscription-based, zero-config
- **OpenAI Codex** — ChatGPT OAuth, uses Codex models
- **Google Gemini (OAuth)** — free tier supported, no API key needed
- **MiniMax (OAuth)** — browser login, no API key needed
- **Qwen OAuth** — reuses local Qwen CLI login

### Custom endpoint

For any OpenAI-compatible API (vLLM, SGLang, Ollama, proxies):

```yaml
provider: custom
base_url: https://your-endpoint.com/v1
api_key: your-key
```

## Configure a provider

### Option 1: Setup wizard

```bash
solviora setup
```

### Option 2: config.yaml

```yaml
provider: anthropic
model: claude-sonnet-4-20250514
```

### Option 3: Command line

```bash
solviora --provider anthropic --model claude-sonnet-4-20250514
```

### Option 4: Environment variable

```bash
export ANTHROPIC_API_KEY=sk-ant-...
solviora --provider anthropic
```

## Local providers (no API key)

### Ollama

1. [Install Ollama](https://ollama.com/download)
2. Pull a model: `ollama pull llama3`
3. Run Solviora:

```bash
solviora --provider ollama --model llama3
```

### LM Studio

1. [Install LM Studio](https://lmstudio.ai/)
2. Load a model in LM Studio
3. Run Solviora:

```bash
solviora --provider lmstudio --model local-model
```

## Fallback model

Configure a fallback for when the primary model is unavailable:

```yaml
model: gpt-4o
fallback_model: gpt-4o-mini
```

## Multiple providers

Solviora supports credential pools — use multiple API keys for the same provider to distribute load:

```yaml
credential_pool:
  - api_key: sk-...
  - api_key: sk-...
```

## Custom base URL

For compatible providers or proxies:

```yaml
provider: openai
base_url: https://my-proxy.example.com/v1
```
