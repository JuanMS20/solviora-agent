# Configuration Reference

All settings for `~/.solviora/config.yaml` and `~/.solviora/.env`. Settings are documented from the codebase defaults.

## config.yaml

### Model & Provider

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `model` | string | `""` | Default LLM model ID |
| `provider` | string | `""` | Default provider (auto-detected if empty) |
| `providers` | object | `{}` | Per-provider settings (base URLs, timeouts, model lists). Each key is a provider ID. |
| `fallback_providers` | array | `[]` | Ordered list of fallback providers tried when primary fails |
| `credential_pool_strategies` | object | `{}` | Strategies for rotating credentials across keys |
| `base_url` | string | — | Custom API base URL (shortcut for single-provider setups) |
| `api_mode` | string | `chat_completions` | API transport mode (`chat_completions`, `codex_responses`, `anthropic`) |

### Agent

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `agent.max_turns` | number | `90` | Max tool-calling iterations per conversation turn |
| `agent.gateway_timeout` | number | `1800` | Gateway agent inactivity timeout (seconds) |
| `agent.restart_drain_timeout` | number | `60` | Graceful drain timeout for gateway restart (seconds) |
| `agent.api_max_retries` | number | `3` | Max app-level retry attempts for API errors |
| `agent.service_tier` | string | `""` | Service tier for OpenAI etc. |
| `agent.tool_use_enforcement` | string/bool | `auto` | Tool-use guidance injection (`auto`, `true`, `false`, or model name list) |
| `agent.gateway_timeout_warning` | number | `900` | Warning threshold before timeout (seconds) |
| `agent.gateway_notify_interval` | number | `180` | "Still working" notification interval (seconds) |
| `agent.gateway_auto_continue_freshness` | number | `3600` | Auto-continue note freshness window (seconds) |
| `agent.image_input_mode` | string | `auto` | Image presentation mode (`auto`, `native`, `text`) |

### Terminal

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `terminal.backend` | string | `local` | Terminal backend (`local`, `docker`, `ssh`, `modal`, `daytona`, `singularity`, `vercel_sandbox`) |
| `terminal.cwd` | string | `.` | Default working directory |
| `terminal.timeout` | number | `180` | Per-command timeout (seconds) |
| `terminal.env_passthrough` | array | `[]` | Env var names to forward to sandboxed execution |
| `terminal.persistent_shell` | bool | `true` | Keep a long-lived bash shell across commands |
| `terminal.docker_image` | string | `nikolaik/python-nodejs:python3.11-nodejs20` | Docker container image |
| `terminal.docker_forward_env` | array | `[]` | Env vars to forward into Docker containers |
| `terminal.docker_env` | object | `{}` | Explicit env var key=value pairs for Docker |
| `terminal.docker_volumes` | array | `[]` | Host directory mounts (`host:container` syntax) |
| `terminal.docker_mount_cwd_to_workspace` | bool | `false` | Mount host cwd into `/workspace` |
| `terminal.docker_run_as_host_user` | bool | `false` | Run container as host user's uid:gid |
| `terminal.singularity_image` | string | — | Singularity container image |
| `terminal.modal_image` | string | — | Modal container image |
| `terminal.daytona_image` | string | — | Daytona container image |
| `terminal.vercel_runtime` | string | `node24` | Vercel Sandbox runtime (`node24`, `node22`, `python3.13`) |
| `terminal.container_cpu` | number | `1` | CPU cores for container backends |
| `terminal.container_memory` | number | `5120` | Memory limit in MB |
| `terminal.container_disk` | number | `51200` | Disk limit in MB |
| `terminal.container_persistent` | bool | `true` | Persist filesystem across sessions |

### Browser

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `browser.inactivity_timeout` | number | `120` | Browser inactivity timeout (seconds) |
| `browser.command_timeout` | number | `30` | Per-command timeout (seconds) |

### Checkpoints

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `checkpoints.enabled` | bool | `true` | Enable filesystem snapshots before destructive ops |
| `checkpoints.max_snapshots` | number | `50` | Max checkpoints per directory |
| `checkpoints.auto_prune` | bool | `false` | Auto-delete stale checkpoints at startup |
| `checkpoints.retention_days` | number | `7` | Stale checkpoint retention |
| `checkpoints.delete_orphans` | bool | `true` | Delete checkpoints for removed directories |

### Compression

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `compression.enabled` | bool | `true` | Enable context compression |
| `compression.threshold` | number | `0.50` | Compress when context exceeds this ratio |
| `compression.target_ratio` | number | `0.20` | Fraction of threshold to preserve as recent tail |
| `compression.protect_last_n` | number | `20` | Minimum recent messages to keep uncompressed |
| `compression.hygiene_hard_message_limit` | number | `400` | Force-compress at this message count |

### Logging

## .env variables

Only store secrets (API keys, tokens, passwords) in `.env`. All other settings go in `config.yaml`.

Key env vars by category:

| Variable | Category | Description |
|----------|----------|-------------|
| `OPENAI_API_KEY` | Provider | OpenAI API key |
| `ANTHROPIC_API_KEY` | Provider | Anthropic API key |
| `GOOGLE_API_KEY` | Provider | Google AI Studio Gemini API key |
| `DEEPSEEK_API_KEY` | Provider | DeepSeek API key |
| `OPENROUTER_API_KEY` | Provider | OpenRouter API key |
| `HF_TOKEN` | Provider | Hugging Face token |
| `XAI_API_KEY` | Provider | xAI (Grok) API key |
| `NVIDIA_API_KEY` | Provider | NVIDIA NIM API key |
| `GLM_API_KEY` | Provider | Z.AI / GLM API key |
| `KIMI_API_KEY` | Provider | Kimi / Moonshot API key |
| `MINIMAX_API_KEY` | Provider | MiniMax API key |
| `DASHSCOPE_API_KEY` | Provider | Alibaba Cloud DashScope API key |
| `XIAOMI_API_KEY` | Provider | Xiaomi MiMo API key |
| `TOKENHUB_API_KEY` | Provider | Tencent TokenHub API key |
| `ARCEEAI_API_KEY` | Provider | Arcee AI API key |
| `KILOCODE_API_KEY` | Provider | Kilo Code API key |
| `GITHUB_TOKEN` / `GH_TOKEN` | Provider | GitHub Copilot token |
| `EXA_API_KEY` | Tool | Exa search API key |
| `FIRECRAWL_API_KEY` | Tool | Firecrawl API key |
| `TERMINAL_SSH_HOST` | Tool | SSH backend host |
| `TERMINAL_SSH_USER` | Tool | SSH backend user |
| `SOLVIORA_HOME` | Runtime | Config directory override (default: `~/.solviora/`) |
| `DISCORD_BOT_TOKEN` | Messaging | Discord bot token |
| `SLACK_BOT_TOKEN` | Messaging | Slack bot token |

:::warning
Non-secret settings (timeouts, thresholds, feature flags) belong in `config.yaml`, not `.env`.
:::

## Profiles

Each profile at `~/.solviora/profiles/<name>/` has its own `config.yaml` and `.env`. Select with `-p`:

```bash
solviora -p work
```
